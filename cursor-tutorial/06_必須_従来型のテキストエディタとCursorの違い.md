# 従来型のテキストエディタとCursorの違い

## はじめに

この章では、**従来型のテキストエディタ**（メモ帳、Notepad++、VS Codeの基本機能など）と**Cursor**の違いを、マーケティング担当者やノンエンジニア向けに整理します。

Cursorは「テキストを編集する」という基本機能は同じですが、**AIがプログラムの実行やデータ処理まで支援する**点が大きな違いです。この違いを理解することで、Cursorをより効果的に活用できるようになります。

## 📊 この章の重要度：🔴 必須

**マーケティング・仕様検討でCursorを活用するために：**
- 従来型エディタでは手作業だった部分を、Cursorではどこまで自動化できるか理解する
- 習得目安：データ分析やレポート作成をCursorで試す前に

## 読者がこれを知ると変わること

**データ分析の場面で：**
- 以前：「BigQueryの結果をダウンロードして、Excelで手動整形」
- 今後：「Cursorに依頼して、クエリ実行→可視化→mdレポートまで一気に作成」

**リサーチの場面で：**
- 以前：「ブラウザで検索して、結果を手でまとめる」
- 今後：「CursorのWeb検索機能で最新情報を取得し、要約まで依頼」

**作業の効率化で：**
- 以前：「同じ作業を毎回手動で繰り返している」
- 今後：「Cursorに自動化スクリプトを書いてもらい、実行まで任せる」

---

## 共通する点：テキストエディタとしての基本機能

従来型のテキストエディタとCursorは、どちらも次の基本機能を持っています。

- プレーンテキストの編集（文字の追加・削除・修正）
- 複数ファイルの同時表示（タブでの切り替え）
- 検索・置換
- シンタックスハイライト（色分け表示）
- フォルダ（プロジェクト）単位でのファイル管理

このため、**従来型エディタの使い方に慣れていれば、Cursorの基本操作もほぼそのまま使えます**。

---

## 決定的な違い：AIが「実行」まで行えるかどうか

### 従来型エディタ：コードを「書く・読む」だけ

従来型のテキストエディタでは、PythonやJavaScriptなどのコードを**書いたり読んだり**することはできますが、そのコードを**実行**する機能は持っていません。

実行したい場合は、ターミナル（黒い画面）を別に開き、自分でコマンドを入力する必要があります。

```
例：Pythonスクリプトを実行する場合
1. エディタで code.py を書く
2. ターミナルを開く
3. 「python code.py」と自分で入力して実行
```

プログラミングに慣れていない方は、この「ターミナルで実行する」ステップでつまずきやすいです。

### Cursor：AIがターミナル操作まで代行

Cursorでは、**AIに「この処理を実行して」と依頼すると、AIがターミナルを操作し、プログラムを実行**してくれます。

- コードを書く
- ターミナルで実行する
- 結果を確認する

この一連の流れを、**自然な日本語で依頼するだけで**AIが行ってくれます。ノンエンジニアでも、指示の出し方さえわかれば、プログラムの実行まで完結できます。

## Cursorでできることの具体例

### 1. BigQueryクエリ実行→可視化→mdレポート作成

**依頼例：**
「BigQueryから先月の売上データを取得して、商品別の棒グラフを作成し、分析結果を `report.md` として出力して」

CursorのAIは、おおむね次の流れで作業します。

1. Python用のBigQuery接続・クエリ実行のコードを書く
2. 取得したデータをグラフ化するコードを追加する（matplotlib 等を使用）
3. ターミナルで Python を実行する
4. グラフ画像を保存する
5. 分析結果をまとめた md ファイルを作成する

※ 実際には、BigQueryへの接続設定（認証情報など）が事前に必要です。環境構築は別途必要になる場合があります。

```mermaid
flowchart LR
    A["依頼"] --> B["Pythonコード生成"]
    B --> C["BigQueryクエリ実行"]
    C --> D["データ可視化"]
    D --> E["report.md 出力"]
```

### 2. CSVの読み込みと作業の自動化

**依頼例：**
「このCSVファイルを読み込んで、〇〇の条件でフィルタし、集計結果を別のCSVに出力して」

- CSVの読み込み
- 条件によるフィルタリング
- 集計・集約
- 結果のファイル出力

これらを、AIがPython等のコードで実装し、実行まで行います。毎月同じ処理をしている定型作業なら、**一度スクリプトを作成すれば、以降は実行するだけで済む**ようになります。

### 3. Web検索を活用した情報収集

Cursorには**Web検索機能**があります。

- **@Web** を指定してチャットすると、AIがインターネット上で検索し、最新情報を回答に反映できる
- マーケティングリサーチ、競合調査、技術トレンドの確認などに使える

従来型エディタには、このような「最新のWeb情報を取得して回答に反映する」機能はありません。

### 4. 繰り返し作業の自動化

手作業で行っていた次のような処理を、スクリプト化して自動実行できます。

- 複数ファイルの一括変換・リネーム
- ログファイルの定期集計
- データの定期バックアップ・整形

「毎週金曜日に〇〇の集計をして、結果をメールで送る」といったルーティンも、AIにスクリプトを作成・実行してもらうことで自動化できる可能性があります。

## 表で比較：従来型エディタ vs Cursor

| 機能 | 従来型テキストエディタ | Cursor |
|------|------------------------|--------|
| テキストの編集 | ✅ | ✅ |
| 検索・置換 | ✅ | ✅ |
| シンタックスハイライト | ✅ | ✅ |
| フォルダ（プロジェクト）管理 | ✅ | ✅ |
| コードの説明・要約 | ❌ | ✅（AIが回答） |
| コードの生成 | ❌ | ✅（AIが生成） |
| プログラムの実行 | ❌（手動でターミナル操作） | ✅（AIがターミナル操作を代行） |
| Web検索 | ❌ | ✅（@Web等） |
| データ取得→可視化→レポート | ❌ | ✅（環境構築は別途必要） |
| CSV処理の自動化 | ❌ | ✅（AIがスクリプト作成・実行） |
| 作業の自動化 | ❌ | ✅（スクリプト作成・実行をAIに依頼） |

---

## 注意点：環境構築は別途必要になることがある

CursorのAIがプログラムを実行できるといっても、**実行環境が整っていないと動きません**。

- **Python** を使う場合は、PCに Python がインストールされている必要がある
- **BigQuery** を使う場合は、Google Cloud の認証設定が必要
- 必要なライブラリ（パッケージ）がインストールされていないと、エラーになる

そのため、**初めて使うときは、開発者やIT担当者に環境構築を依頼する**ことがあります。一方で、Python がすでにインストールされていて、CSVの読み込みや簡単な集計だけであれば、Cursor単体でも始められる場合があります。

## まとめ

### この章で学んだこと

1. **従来型エディタ**は「書く・読む」に特化。実行は自分でターミナルを操作する必要がある
2. **Cursor**では、AIがターミナル操作まで代行するため、**プログラムの実行**や**データ処理の自動化**が可能
3. BigQueryクエリ→可視化→mdレポート、CSV処理、Web検索などを、自然な日本語で依頼して実現できる
4. 本格的に使うには、Python や BigQuery などの環境構築が事前に必要になる場合がある
